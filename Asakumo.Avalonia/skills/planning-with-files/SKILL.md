# Planning with Files

解决 AI 代理的三大问题：
1. **易失性记忆** — 上下文重置后信息丢失
2. **目标漂移** — 多次工具调用后忘记原始目标
3. **错误隐藏** — 失败未被追踪，重复相同错误

## 核心理念

```
上下文窗口 = RAM（易失、有限）
文件系统 = 磁盘（持久、无限）
→ 任何重要内容都写入磁盘
```

## 三文件模式

每个复杂任务创建三个文件：

| 文件 | 用途 | 更新时机 |
|------|------|----------|
| `task_plan.md` | 跟踪阶段和进度 | 每完成一步 |
| `findings.md` | 存储研究和发现 | 每2次操作后 |
| `progress.md` | 会话日志和测试结果 | 每次会话结束 |

## 快速开始

### 1. 初始化规划文件

根据模板创建三个文件：

```
templates/task_plan.md   → 复制到项目根目录
templates/findings.md    → 复制到项目根目录
templates/progress.md    → 复制到项目根目录
```

### 2. 定义任务计划

编辑 `task_plan.md`：
- 填写 Goal（一句话描述目标）
- 分解为多个 Phase（阶段）
- 每个 Phase 分解为多个 Step（步骤）

### 3. 执行并跟踪

- 按步骤执行
- 完成 Step 后立即勾选 `[x]`
- 每2次操作后保存发现到 `findings.md`
- 记录所有错误和解决方法

### 4. 会话结束

更新 `progress.md`：
- 记录本次会话的 Actions
- 列出修改的文件
- 记录测试结果
- 记录错误和解决方法
- 写明下一步

## 核心规则

### 2-行动规则
每进行2次查看/浏览器操作后，必须保存发现到 `findings.md`。

### 错误记录规则
记录所有错误：
```markdown
| 错误 | 原因 | 解决方法 |
|------|------|----------|
| [错误信息] | [为什么发生] | [如何修复] |
```

### 不重复失败
- 跟踪尝试的方法
- 失败后变换方法
- 永不重复相同错误

## 模板文件

- `templates/task_plan.md` - 任务计划模板
- `templates/findings.md` - 发现记录模板
- `templates/progress.md` - 进度日志模板

## 适用场景

**适用：**
- 多步骤任务（3+ 步骤）
- 研究任务
- 构建/创建项目
- 跨多次工具调用的任务

**不适用：**
- 简单问题
- 单文件编辑
- 快速查询

## 完成检查清单

任务完成前验证：
- [ ] `task_plan.md` 所有步骤已勾选
- [ ] `findings.md` 包含关键发现
- [ ] `progress.md` 已更新会话日志
- [ ] 所有错误已记录解决方法
- [ ] 未完成项已记录在 Next Steps
