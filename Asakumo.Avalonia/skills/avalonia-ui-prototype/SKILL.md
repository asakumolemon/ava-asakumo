# Avalonia UI Prototype Skill

快速为 Avalonia 应用生成 ASCII 线框图原型，通过多轮对话迭代完善设计。

## When to Use

使用此 skill 当需要：
1. 快速可视化页面布局概念
2. 在编写 AXAML 前验证 UI 结构
3. 与团队讨论界面设计时提供参考
4. 规划 Avalonia 应用的导航流程
5. 探索不同布局方案（如列表 vs 网格）

## Interaction Flow

采用**多轮交互**模式：

```
用户: 帮我设计一个聊天页面
   ↓
Skill: 确认几个问题：
       1. 目标平台？（手机/平板/桌面）
       2. 是否需要侧边栏显示会话列表？
       3. 输入框需要哪些功能？（文字/图片/语音）
   ↓
用户: 手机端，需要会话列表，支持文字和图片
   ↓
Skill: [生成 ASCII 原型]
   ↓
用户: 调整：输入框放在底部固定
   ↓
Skill: [更新 ASCII 原型]
```

## Output Format

使用 **Box-drawing 字符**绘制线框图：

```
┌──────────────────────────────────────┐
│ [Title]                         [⚙]  │ ← 标题栏
├──────────────────────────────────────┤
│                                      │
│ ┌──────────────────────────────┐     │
│ │[Icon] Item 1          [>]   │     │ ← 列表项
│ ├──────────────────────────────┤     │
│ │[Icon] Item 2          [>]   │     │
│ └──────────────────────────────┘     │
│                                      │
│                              [+ FAB] │ ← 浮动按钮
└──────────────────────────────────────┘
```

### 符号说明

| 符号 | 含义 |
|------|------|
│ `┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼` | 边框角 |
| `─ │` | 水平和垂直线 |
| `[...]` | 按钮或可交互元素 |
| `(...)` | 输入框或文本区域 |
| `<--` | 注释说明 |

## Layout Guidelines

### 手机端 (Phone)
- 单列布局为主
- 底部导航或汉堡菜单
- 全宽列表页

### 平板端 (Tablet)
- 主从布局 (Master-Detail)
- 侧边栏可固定
- 网格视图更合适

### 桌面端 (Desktop)
- 多列布局
- 固定导航栏
- 窗口可调整大小适配

## Avalonia-Specific Tips

### 1. Compiled Bindings
```
在原型中标注绑定路径：
┌──────────────────┐
│ {Binding Title} │ ← 数据绑定
└──────────────────┘
```

### 2. Styles & Themes
```
标注主题色：
┌──────────────────┐
│ [Primary Button] │ ← AccentColor
│ [Secondary Btn]  │ ← SecondaryColor
└──────────────────┘
```

### 3. Responsive Layout
```
桌面端：
┌────────────────────────────────────────┐
│ [侧边栏]  │       [主内容区]          │
└────────────────────────────────────────┘

移动端：
┌──────────────────┐
│    [主内容区]    │
└──────────────────┘
```

## Best Practices

1. **保持简洁** - 不在原型中包含过多细节
2. **明确交互** - 标注可点击区域
3. **平台适配** - 考虑单手操作区域
4. **迭代优化** - 先生成基础布局再细化

## Limitations

- 仅支持静态布局展示
- 无法演示动画
- 不支持复杂手势交互的展示
- 响应式布局需分别绘制
